# 数据库设计

当前项目使用轻量文件存储模拟核心数据表，便于课程演示和本地部署。

## User

存储位置：`runtime/users.json`

- `username`：用户名。
- `password_hash`：Werkzeug 哈希后的密码。
- `role`：`normal_user`、`researcher`、`data_manager`、`admin`。
- `status`：`active` 或 `disabled`。
- `created_at`：创建时间。

说明：公开注册默认创建普通用户；仅在用户库为空时允许首个账号作为管理员引导创建。管理员可修改角色、启用/禁用或删除用户。

## Dataset_Info

存储位置：`data/registry.json`

- `dataset_id`：数据集唯一标识。
- `name`：数据集名称。
- `data_path`：数据文件路径。
- `source`：数据来源，例如 `local` 或 `upload`。
- `description`、`species`、`tissue`：可编辑元信息。
- `status`：`registered`、`validated`、`loaded`、`offline`、`error`。
- `cell_count`、`vector_dim`：校验后记录的数据规模。
- `embedding_method`：默认 `obsm/X_pca`。
- `visualization_method`：默认 `obsm/X_umap`。
- `metadata_fields`：可用于过滤和展示的元数据字段。
- `sample_cell_ids`：前端展示和演示辅助 Cell ID。
- `created_at`、`updated_at`：登记和更新时间。

## Cell_Vector

细胞向量不单独落库，运行时从 `.h5ad` 读取：

- 检索向量：`obsm/X_pca`。
- 可视化坐标：`obsm/X_umap`。
- 细胞 ID：`obs/_index`。
- 元信息：`cell_type`、`disease`、`AgeGroup`、`tissue`。

## Index_Info

存储位置：`indexes/*.faiss` 和 `indexes/*.meta.json`

`.faiss` 保存 FAISS CPU 索引，`.meta.json` 保存：

- `index_id`、`index_type`、`metric`、`build_mode`。
- `dataset_ids`、`vector_count`、`dimension`。
- IVF 参数：`nlist`、`nprobe`。
- HNSW 参数映射：`nprobe` 字段记录 `efSearch`，构建参数在前端状态和索引摘要中展示。
- `index_path`、`metadata_path`、`build_duration_ms`。
- `index_to_cell`：FAISS 返回下标到 `dataset_id + row_index` 的映射。

## Query_Log

存储位置：`logs/query_log.jsonl`

- `created_at`：日志时间。
- `query_type`：`ann`、`exact`、`vector` 等。
- `query_object`：Cell ID 或向量维度描述。
- `dataset_id`、`index_id`、`top_k`。
- `metadata_filters`：条件过滤。
- `latency_ms`、`result_count`。
- `status`、`error`。

## Benchmark_Result

存储位置：`logs/benchmark_results.jsonl`

- `benchmark_type`：`compare` 或 `batch`。
- `index_id`、`dataset_id`、`top_k`。
- `recall`、`overlap_count`、`ann_latency_ms`、`exact_latency_ms`、`speedup`。
- `total_queries`、`successful`、`failed`、`qps`。
- `latency_avg_ms`、`latency_p50_ms`、`latency_p99_ms`。

## LLM_Log

存储位置：`logs/llm_analysis_log.jsonl`

- `created_at`、`provider`、`model`。
- `latency_ms`、`attempts`、`cached`。
- `status`、`error`。

## 设计取舍

本项目不引入完整关系型数据库，原因是课程验收重点在 ANN 检索链路、数据集/索引生命周期和可视化分析。文件存储降低部署复杂度，同时保留了可迁移到数据库表的字段结构。

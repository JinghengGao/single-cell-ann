# 系统测试

## 测试环境

- 后端：Python、Flask、pytest。
- 前端：Node.js、npm、Vite。
- 默认数据集：`data/liver.h5ad`。
- 默认索引目录：`indexes/`。

## 自动化测试

运行：

```powershell
python -m pytest tests\test_backend_api.py
```

当前结果：

```text
21 passed, 2 skipped
```

跳过原因：默认 Python 环境未安装 FAISS 时，FAISS 集成索引测试会跳过。

覆盖范围：

- 健康检查和 CORS。
- 用户注册、登录、退出、权限控制和账号启停。
- 数据集扫描、校验、加载、元信息编辑、下线、恢复和删除。
- 索引构建、加载、删除和状态查询。
- Cell ID 检索、向量检索、条件过滤、ANN vs Exact 和批量评测。
- 查询日志和评测日志接口。
- LLM 分析请求校验、错误处理、重试、缓存和本地 Provider。
- UMAP 过滤、统计和基因表达叠加。

## 烟测建议

```powershell
python tests\smoke_midterm.py
```

如果当前 Python 环境没有 FAISS，烟测会跳过索引/检索部分。若使用 conda 环境，请先单独确认 FAISS IVF 训练在本机稳定。

## 手工验收路径

1. 启动后端和前端。
2. 登录管理员或研究人员账号。
3. 扫描并校验 `liver` 数据集。
4. 加载数据集，查看细胞数和向量维度。
5. 加载已有 `liver_ivf_flat` 索引，或构建 HNSW 索引。
6. 输入样例 Cell ID 执行 Top-K 检索。
7. 添加 `cell_type` 过滤，验证条件检索。
8. 开启 ANN vs Exact，查看 Recall、ANN 延迟、Exact 延迟和 speedup。
9. 执行批量评测，查看 QPS、Avg、P50、P99。
10. 点击 UMAP 点，确认右侧查询 Cell ID 自动回填。
11. 使用 `ALB` 做基因表达叠加。
12. 打开 AI 辅助分析，基于检索结果生成报告。
13. 在系统管理页查看查询日志和评测日志。

## 已知风险

- 当前本机 conda 环境中 FAISS IVF `train()` 可能进程级崩溃。
- 演示前建议提前构建并保存索引，或使用 HNSW/CPU FAISS 环境。
- 多数据集联合检索需要至少两个向量维度一致的 `.h5ad` 文件；当前默认数据只包含 `liver.h5ad`。

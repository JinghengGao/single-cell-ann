# 需求分析与系统设计

## 用户角色

- 访客：浏览首页、公开数据状态、索引状态和只读可视化。
- 普通用户：执行 Top-K 检索、精确检索、ANN 对比、批量评测和可视化查看。
- 研究人员：加载数据集，构建、加载、切换、删除索引，执行检索和评测。
- 数据维护者：扫描、上传、校验、维护数据集，并可加载数据集和参与索引管理。
- 管理员：拥有全部权限，负责用户、角色、账号状态、日志和系统状态监控。

## 功能需求

- 认证与权限：注册、登录、退出、Token 鉴权、角色校验、用户列表、角色修改、账号启停和删除。
- 数据集管理：扫描 `.h5ad`、上传、校验、加载、激活、下线、恢复、删除和元信息编辑。
- 索引管理：构建 IVF_FLAT/HNSW，支持联合/独立索引、L2/Cosine/IP 度量、索引保存、加载、切换和删除。
- 检索服务：Cell ID 检索、向量检索、Top-K、元数据条件过滤、精确检索、ANN vs Exact 对比和批量检索。
- 可视化服务：UMAP 抽样、元信息着色、条件过滤、基因表达叠加、Top-K 高亮和点击点回填查询。
- RAG 与 AI 分析：自然语言问题可驱动数据库检索并生成回答；已有 Top-K 检索结果也可交给 LLM 做解释，支持重试、缓存和日志。
- 监控审计：健康检查、FAISS 状态、数据集/索引状态、查询日志和评测日志。

## 系统架构

系统采用前后端分离结构。前端负责工作台 UI、表单状态、图表交互和结果展示；后端负责 API、认证、`.h5ad` 读取、FAISS 索引、检索、评测、日志和 LLM 调用。

主要目录：

- `backend/routes/`：HTTP API 路由。
- `backend/services/`：认证、数据、索引、检索、可视化和 LLM 业务逻辑。
- `frontend/src/pages/`：首页、登录、分析、AI、数据集、索引和管理页面。
- `data/`：演示数据和数据集登记表。
- `indexes/`：索引文件和索引元数据。
- `logs/`：查询日志、评测日志和 LLM 日志。
- `runtime/`：用户文件和运行状态。

## 关键流程

1. 用户登录后获得 Bearer Token。
2. 数据维护者扫描或上传 `.h5ad` 数据集。
3. 系统校验 `obsm/X_pca`、`obsm/X_umap`、`obs/_index` 和元数据字段。
4. 研究人员选择一个或多个数据集构建 IVF_FLAT/HNSW 索引。
5. 用户输入 Cell ID 或向量，设置 Top-K 和条件过滤后发起检索。
6. 后端返回命中细胞、距离、相似度、元数据、UMAP 坐标和查询耗时。
7. 前端在 UMAP 中高亮查询细胞与 Top-K 命中，并展示统计和评测指标。
8. 用户可进一步执行 ANN vs Exact、批量评测、自然语言 RAG 或已有结果 AI 辅助分析。

## 非功能需求

- 可运行性：后端和前端均可在本地启动。
- 可演示性：默认数据集和已有索引可支持课堂演示。
- 可扩展性：数据集、索引类型、LLM Provider 和日志文件均可扩展。
- 安全边界：本项目为课程演示系统，不作为生产级身份认证或数据库系统。

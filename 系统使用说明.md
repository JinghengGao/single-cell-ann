# 单细胞 ANN 检索系统使用说明

本文说明当前系统有哪些功能、如何导入数据集、如何构建索引、如何查询相似细胞，以及 `cell_id` 从哪里获取。

## 1. 当前功能

系统页面明确分为五个模块：

- 用户信息模块：未登录可预览主页；登录后按身份开放操作，支持本地演示账号的注册、登录、退出和当前用户显示。
- 数据管理模块：扫描本地 `.h5ad`、上传 `.h5ad`、校验数据集、选择一个或多个数据集。
- 索引构建模块：使用 FAISS IVF_FLAT 构建 ANN 索引，支持联合索引和独立索引。
- 查询检索模块：输入 `cell_id` 和 Top-K，返回相似细胞及元信息。
- 可视化展示模块：UMAP 分析工作台，支持元信息着色、过滤、统计摘要、基因表达叠加、Top-K 连线、点击选点和 CSV 导出。

核心数据字段：

- 检索向量：`obsm["X_pca"]`
- 可视化坐标：`obsm["X_umap"]`
- 细胞 ID：`obs["_index"]`
- 元信息：`cell_type`、`disease`、`AgeGroup`、`tissue`

## 2. 权限说明

当前系统不是生产级权限系统，但前端和后端都会按演示身份限制操作。

- 未登录：可以预览主页、查看模块布局、查看公开状态，但不能扫描、上传、校验、加载、构建索引或查询。
- 普通用户：可以查询 Top-K 相似细胞和查看可视化。
- 研究人员：可以加载数据集、构建/切换索引、查询和查看可视化。
- 数据维护者：可以扫描、上传、校验数据集，也可以加载数据集、构建索引和查询。
- 管理员：拥有全部演示权限。

## 3. 启动系统

后端：

```powershell
conda activate single-cell-ann
python -m backend.app
```

前端：

```powershell
cd frontend
npm install
npm run dev
```

浏览器打开 Vite 输出的地址，通常是：

```text
http://127.0.0.1:5173
```

## 4. 数据集导入

系统支持两种导入方式。

本地目录扫描：

```text
data/liver.h5ad
data/datasets/*.h5ad
```

把 `.h5ad` 文件放入上述目录后，在页面点击“扫描本地”。默认演示数据 `data/liver.h5ad` 会被登记为 `dataset_id = liver`。

网页上传：

在“数据管理模块”选择 `.h5ad` 文件并点击“上传”。文件会保存到：

```text
data/uploads/
```

上传文件不会提交 Git。

## 5. 数据集校验

选择数据集后点击“校验选中”。系统会检查：

- 是否存在 `obsm/X_pca`
- 是否存在 `obsm/X_umap`
- 是否存在 `obs/_index`
- PCA 向量行数、UMAP 行数、cell_id 数量是否一致

校验成功后页面会显示细胞数、向量维度、元信息字段和示例 `cell_id`。默认 `liver.h5ad` 应显示：

```text
cell_count = 69032
vector_dim = 30
```

## 6. 索引构建

在“索引构建模块”中选择数据集和构建模式。

联合索引：

- 选择一个或多个数据集。
- 系统把这些数据集的 PCA 向量合并成一个 FAISS 索引。
- 查询结果会返回每个命中细胞所属的数据集。
- 多数据集联合索引要求 PCA 维度一致。

独立索引：

- 选择一个或多个数据集。
- 系统为每个数据集分别构建一个 FAISS 索引。
- 可以在索引下拉框中切换当前索引。

默认参数：

```text
index_type = IVF_FLAT
metric = l2
nlist = 256
nprobe = 16
```

## 7. 查询相似细胞

在“查询检索模块”中：

1. 选择查询数据集，或使用自动匹配。
2. 输入 `cell_id`。
3. 设置 Top-K。
4. 点击“查询”。

返回结果包括：

- 排名
- 所属数据集
- `cell_id`
- 距离
- 相似度
- `cell_type`
- `disease`
- `AgeGroup`
- `tissue`
- UMAP 坐标
- 查询耗时

如果多个数据集里存在相同 `cell_id`，需要指定查询数据集，避免歧义。

## 8. cell_id 示例

`cell_id` 来自 `.h5ad` 文件的：

```text
obs["_index"]
```

默认 `liver.h5ad` 可直接测试这些 ID：

```text
AAACCTGAGCAGGTCA-1_2
AAACCTGAGGCCATAG-1_2
AAACCTGCAAGCTGAG-1_2
AAACCTGGTCCAGTTA-1_2
AAACCTGTCATAAAGG-1_2
AAACCTGTCTGGAGCC-1_2
AAACGGGAGCCAGGAT-1_2
AAACGGGAGCGAAGGG-1_2
AAACGGGAGTCCGGTC-1_2
AAACGGGAGTTACCCA-1_2
```

也可以通过命令查看更多：

```powershell
conda activate single-cell-ann
python -c "import h5py; f=h5py.File('data/liver.h5ad','r'); print([x.decode() for x in f['obs']['_index'][:50]])"
```

## 9. 可视化分析工作台

可视化展示模块现在支持以下分析操作：

- 着色：按数据集、细胞类型、疾病、年龄组、组织或基因表达量着色。
- 过滤：按 `cell_type`、`disease`、`AgeGroup`、`tissue` 筛选 UMAP 点。
- 基因叠加：输入基因名或 Ensembl ID，例如 `ALB`，点击“基因上色”。
- 查询联动：查询后自动高亮查询细胞和 Top-K 命中细胞，并用虚线连接。
- 图上选点：点击 UMAP 上的细胞，会把该细胞填入查询表单。
- 统计面板：显示可见细胞数、抽样点数、抽样比例、类别组成和表达量摘要。
- 导出：点击“导出 CSV”保存当前 UMAP 点和元信息。

后端读取基因表达时只读取当前抽样点在 CSR 稀疏矩阵中的单基因表达值，不会把完整表达矩阵常驻内存。

## 10. API 示例

如果还没有账号，先在页面注册，或调用注册接口。随后登录获取 token：

```powershell
$login = curl -Method POST http://127.0.0.1:5000/api/auth/login `
  -ContentType "application/json" `
  -Body "{\"username\":\"admin_user\",\"password\":\"secret123\"}" | ConvertFrom-Json
$token = $login.token
```

扫描数据集：

```powershell
curl -X POST http://127.0.0.1:5000/api/datasets/scan `
  -H "Authorization: Bearer $token"
```

校验数据集：

```powershell
curl -X POST http://127.0.0.1:5000/api/datasets/validate `
  -H "Authorization: Bearer $token" `
  -H "Content-Type: application/json" `
  -d "{\"dataset_ids\":[\"liver\"]}"
```

构建联合索引：

```powershell
curl -X POST http://127.0.0.1:5000/api/index/build `
  -H "Authorization: Bearer $token" `
  -H "Content-Type: application/json" `
  -d "{\"dataset_ids\":[\"liver\"],\"mode\":\"combined\",\"nlist\":256,\"nprobe\":16}"
```

构建独立索引：

```powershell
curl -X POST http://127.0.0.1:5000/api/index/build `
  -H "Authorization: Bearer $token" `
  -H "Content-Type: application/json" `
  -d "{\"dataset_ids\":[\"liver\"],\"mode\":\"separate\"}"
```

查询 Top-K：

```powershell
curl -X POST http://127.0.0.1:5000/api/search `
  -H "Authorization: Bearer $token" `
  -H "Content-Type: application/json" `
  -d "{\"dataset_id\":\"liver\",\"cell_id\":\"AAACCTGAGCAGGTCA-1_2\",\"top_k\":5}"
```

获取 UMAP 点：

```powershell
curl "http://127.0.0.1:5000/api/visualization/cells?limit=5000&dataset_ids=liver"
```

获取可视化选项和基因匹配：

```powershell
curl "http://127.0.0.1:5000/api/visualization/options?dataset_ids=liver&gene_query=ALB"
```

获取 ALB 表达叠加的 UMAP 点：

```powershell
curl "http://127.0.0.1:5000/api/visualization/cells?dataset_ids=liver&limit=5000&color_by=gene:ALB"
```

按细胞类型过滤：

```powershell
curl "http://127.0.0.1:5000/api/visualization/cells?dataset_ids=liver&filter_cell_type=hepatocyte&color_by=cell_type"
```

## 12. 常见问题

Network Error：

- 确认后端正在运行。
- 确认前端请求的是 `http://127.0.0.1:5000/api`。
- 如果 Vite 自动切换端口，刷新页面或重启后端。

构建索引失败：

- 先扫描并校验数据集。
- 联合索引的数据集 PCA 维度必须一致。
- 检查 FAISS 是否可用：`python -c "import faiss; print(faiss.get_num_gpus())"`。

查询无结果或报 unknown cell：

- 确认 `cell_id` 来自当前选择的数据集。
- 如果使用独立索引，确认当前索引对应的数据集。

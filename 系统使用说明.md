# 单细胞 ANN 检索系统使用说明

本文面向演示和结项验收，说明系统如何启动、如何管理数据集、如何构建索引、如何执行条件 Top-K 检索、如何查看评测指标和可视化结果。

## 1. 功能概览

系统包含以下模块：

- 首页与登录：展示系统状态、FAISS 状态和 Demo 检索入口；支持注册、登录和访客浏览。
- 数据集管理：扫描、上传、校验、加载、激活、下线、恢复、删除 `.h5ad` 数据集，并维护名称、来源、物种、组织和描述。
- 索引管理：构建 IVF_FLAT 或 HNSW 索引，支持联合索引、独立索引、L2/Cosine/IP 度量、磁盘加载、切换和删除。
- 分析工作台：执行 Cell ID 检索、向量检索、条件过滤、精确检索、ANN vs Exact 对比和批量评测。
- UMAP 可视化：按元数据或基因表达着色，支持过滤、Top-K 高亮、查询点连线、点击点回填查询细胞和 CSV 导出。
- RAG 与 AI 辅助分析：自然语言问题可先自动检索单细胞数据库，再基于检索证据生成回答；也支持对已有 Top-K 结果生成细胞邻域解释报告。
- 系统管理：管理员维护用户、角色、账号状态，并查看查询日志、评测日志和系统状态。

核心数据字段：

- 检索向量：`obsm/X_pca`
- 可视化坐标：`obsm/X_umap`
- 细胞 ID：`obs/_index`
- 元信息：`cell_type`、`disease`、`AgeGroup`、`tissue`

## 2. 权限说明

- 未登录：可浏览首页、数据集列表、索引状态和公开可视化，不可执行检索或写操作。
- 普通用户：可执行检索、对比评估、批量查询和可视化查看。
- 研究人员：可加载数据集、构建/切换/加载/删除索引，并执行检索和评估。
- 数据维护者：可扫描、上传、校验和维护数据集，也可加载数据集、构建索引和检索。
- 管理员：拥有全部权限，并可管理用户和查看监控日志。

首次空用户库可创建管理员账号；已有用户后，公开注册不能直接创建管理员。

## 3. 启动系统

后端：

```powershell
conda activate single-cell-ann
python -m backend.app
```

或使用脚本：

```powershell
.\start_backend_local.ps1
.\stop_backend.ps1
```

前端：

```powershell
cd frontend
npm install
npm run dev
```

浏览器打开 Vite 输出地址，通常是：

```text
http://127.0.0.1:5173
```

## 4. 数据集管理

本地扫描目录：

```text
data/liver.h5ad
data/datasets/*.h5ad
```

网页上传目录：

```text
data/uploads/
```

数据集操作流程：

1. 进入“数据集”页。
2. 点击“扫描本地”或上传 `.h5ad`。
3. 选择数据集并点击“校验选中”。
4. 校验通过后可维护名称、来源、物种、组织和描述。
5. 点击“加载数据集”或“激活”进入当前工作区。
6. 不再使用的数据集可下线、恢复或从登记表删除。

默认 `data/liver.h5ad` 预期：

```text
dataset_id = liver
cell_count = 69032
vector_dim = 30
```

## 5. 索引管理

支持两类索引：

- IVF_FLAT：支持 `nlist`、`nprobe` 参数，适合演示 ANN 参数调优和 ANN vs Exact 对比。
- HNSW：支持 `M`、`efConstruction`、`efSearch` 参数，适合规避部分本机 FAISS IVF 训练不稳定问题。

支持两种构建模式：

- 联合索引：多个数据集向量合并成一个索引，实现跨数据集搜索；要求向量维度一致。
- 独立索引：为每个数据集分别构建索引，可在索引列表中切换。

支持索引生命周期：

- 构建索引并保存到 `indexes/*.faiss`。
- 保存 `indexes/*.meta.json`，记录索引到细胞的映射。
- 通过索引 ID 从磁盘加载。
- 在多个索引之间切换。
- 删除索引文件和内存状态。

注意：当前本机 conda 环境中 FAISS IVF `train()` 在测试进程里存在崩溃风险。演示时建议提前准备已有索引，或使用 HNSW/CPU FAISS 环境。

## 6. 条件 Top-K 检索

在“分析工作台”右侧面板：

1. 选择查询数据集，或保留自动匹配。
2. 输入 `cell_id`，也可以点击 UMAP 点自动回填。
3. 设置 Top-K。
4. 可选填写条件过滤：`cell_type`、`disease`、`AgeGroup`、`tissue`。
5. 点击“执行检索”。

返回结果包括：

- Rank、数据集、Cell ID。
- 距离、相似度。
- `cell_type`、`disease`、`AgeGroup`、`tissue`。
- UMAP 坐标。
- 查询耗时 `query_time_ms`。

向量检索：

- 开启“查询向量输入”。
- 输入与索引维度一致的数字向量。
- 系统调用 `/api/search/vector` 返回 Top-K。

## 7. 实验评估

系统支持三类评估证据：

- 单次检索耗时：每次 ANN、Exact 或向量检索均返回 `query_time_ms`。
- ANN vs Exact：调用 `/api/search/compare`，返回 Recall、ANN 延迟、Exact 延迟、命中重叠数和 speedup。
- 批量评测：调用 `/api/search/batch`，返回总耗时、成功数、失败数、QPS、Avg、P50、P99、Min、Max。

日志位置：

```text
logs/query_log.jsonl
logs/benchmark_results.jsonl
```

管理员可在“系统管理”页查看最近查询日志和评测日志。

## 8. 可视化使用

UMAP 工作台支持：

- 着色：数据集、细胞类型、疾病、年龄组、组织或基因表达。
- 过滤：按 `cell_type`、`disease`、`AgeGroup`、`tissue` 筛选。
- 基因叠加：输入基因名或 Ensembl ID，例如 `ALB`。
- 检索联动：查询后高亮查询细胞和 Top-K 命中点，并绘制连线。
- 图上选点：点击 UMAP 细胞点，右侧查询表单自动回填 Cell ID。
- 导出：导出当前 UMAP 点及元数据为 CSV。

## 9. RAG 与 AI 辅助分析

AI 分析依赖 `.env` 中的 LLM 配置。常见配置：

```bash
SCANN_LLM_PROVIDER=siliconflow
SCANN_LLM_API_KEY=sk-...
SCANN_LLM_MODEL=Qwen/Qwen3-8B
```

支持 OpenAI-compatible 本地服务：

```bash
SCANN_LLM_PROVIDER=local
SCANN_LLM_API_URL=http://127.0.0.1:11434/v1/chat/completions
SCANN_LLM_MODEL=qwen3:8b
```

自然语言 RAG 使用方式：

1. 确认已有 ready 状态的索引。
2. 打开“AI 辅助分析”。
3. 在“自然语言 RAG 检索”中输入问题，例如“帮我检索肝细胞中和健康样本相似的 Top-K 细胞，并解释邻域特点”。
4. 系统会抽取 Cell ID 或元数据条件，执行 ANN 检索，再把检索证据交给 LLM 回答。

已有结果分析使用方式：

1. 先在分析工作台执行一次 Top-K 检索。
2. 回到“AI 辅助分析”。
3. 输入分析重点，例如“这些邻居主要是什么细胞类型，有没有异常疾病分布？”。
4. 点击生成分析报告。

## 10. 常用 API 示例

登录：

```powershell
$login = curl.exe -s -X POST http://127.0.0.1:5000/api/auth/login `
  -H "Content-Type: application/json" `
  -d "{\"username\":\"demo_admin\",\"password\":\"demo_password\"}" | ConvertFrom-Json
$token = $login.token
```

扫描与校验：

```powershell
curl.exe -X POST http://127.0.0.1:5000/api/datasets/scan -H "Authorization: Bearer $token"

curl.exe -X POST http://127.0.0.1:5000/api/datasets/validate `
  -H "Authorization: Bearer $token" `
  -H "Content-Type: application/json" `
  -d "{\"dataset_ids\":[\"liver\"]}"
```

加载已有索引：

```powershell
curl.exe -X POST http://127.0.0.1:5000/api/index/load `
  -H "Authorization: Bearer $token" `
  -H "Content-Type: application/json" `
  -d "{\"index_id\":\"liver_ivf_flat\"}"
```

条件检索：

```powershell
curl.exe -X POST http://127.0.0.1:5000/api/search `
  -H "Authorization: Bearer $token" `
  -H "Content-Type: application/json" `
  -d "{\"dataset_id\":\"liver\",\"cell_id\":\"AAACCTGAGCAGGTCA-1_2\",\"top_k\":5,\"filter_cell_type\":\"hepatocyte\"}"
```

对比评估：

```powershell
curl.exe -X POST http://127.0.0.1:5000/api/search/compare `
  -H "Authorization: Bearer $token" `
  -H "Content-Type: application/json" `
  -d "{\"dataset_id\":\"liver\",\"cell_id\":\"AAACCTGAGCAGGTCA-1_2\",\"top_k\":10}"
```

UMAP 点：

```powershell
curl.exe "http://127.0.0.1:5000/api/visualization/cells?dataset_ids=liver&limit=5000&color_by=cell_type"
```

## 12. 常见问题

Network Error：

- 确认后端正在 `127.0.0.1:5000` 运行。
- 确认前端请求的 API 地址正确。
- 强制刷新页面，必要时清理浏览器 `localStorage` 中旧 Token。

点击 UMAP 不回填：

- 已修复为点击点时回填 `cell_id`，并自动关闭向量输入模式。
- 如果页面仍无变化，重启 Vite 或强制刷新。

索引构建失败：

- 先扫描、校验并加载数据集。
- 联合索引要求向量维度一致。
- 如果 IVF 构建导致 FAISS 崩溃，改用 HNSW 或加载已有索引。

查询 unknown cell：

- 确认 Cell ID 来自当前数据集。
- 多数据集存在相同 Cell ID 时，指定查询数据集。
